I'm building an MCP server that lets AI coding agents autonomously test Chrome extensions — reload, interact, inspect errors, and validate behavior without a human in the loop. I built two Chrome extensions with Claude Code recently and the testing loop was entirely manual: reload, click around, check the console, paste errors back. For simple extensions like GitTrack, there's no reason an agent can't do that itself.

recent —

  • Auditex

    A Chrome extension that runs Kokoro TTS entirely in your browser — no backend, no paywall, no server costs. Ever.

    I got frustrated with Speechify's paywall and built a free alternative. The first version used a cloud backend — then I realized I'd just recreated the same cost problem. So I moved the entire TTS pipeline into the browser using ONNX Runtime. 310MB model, running on-device, streaming audio through a chunked buffer with IndexedDB persistence. It works. I wrote about the whole journey.

    JavaScript, ONNX Runtime Web, Web Audio API, Web Workers, IndexedDB, OPFS

  • GitTrack

    A Chrome extension that tracks job applications locally and keeps you accountable through streaks.

    My friends and I were manually texting each other daily application counts to stay accountable. That worked until it didn't. GitTrack injects directly into job listing pages — one click to mark applied, stored locally, instant lookup. Added duplicate detection, streaks, and the ability to share streak counts. Three months in, 1000+ jobs tracked, and we still call each other when someone's about to break their streak.

    JavaScript, Chrome Extensions Manifest V3, IndexedDB
